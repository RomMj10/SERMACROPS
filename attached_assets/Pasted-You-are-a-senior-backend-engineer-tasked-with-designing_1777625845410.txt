You are a senior backend engineer tasked with designing a system for a manufacturing company named SERMACROPS and implement a working Electronic Data Interchange (EDI) system for a manufacturing company.

## 📌 System Overview
Build a complete Node.js-based EDI system that:
- Uses ANSI X12 format for all EDI documents
- Uses AS2 protocol for secure transmission of EDI messages
- Simulates real-world supply chain communication between:
  - Coffee Shop (Client)
  - Raw Materials Supplier
  - Logistics Provider

## 📌 Core Requirements

### 1. Tech Stack
Follows the MERN stack (MongoDB, Express.js, React, and Node.js)
- Node.js (Express.js for API)
- AS2 communication (use libraries or simulate if needed)
- ANSI X12 parsing and generation (use npm libraries or custom parser)
- Axios or similar for HTTP communication
- Environment-based configuration (.env)
- Docker support (optional)
- MongoDB for storing transactions, and inventory

---

### 2. EDI Transactions to Support

#### From Coffee Shop → SERMACROPS
- EDI 850 (Purchase Order)

#### From SERMACROPS → Coffee Shop
- EDI 855 (Purchase Order Acknowledgment)
- EDI 856 (Advance Ship Notice)
- EDI 810 (Invoice)

#### From SERMACROPS → Raw Materials Supplier
- EDI 850 (Purchase Order)

#### From Raw Materials Supplier → SERMACROPS
- EDI 855 (PO Acknowledgment)
- EDI 856 (ASN)
- EDI 810 (Invoice)

#### From SERMACROPS → Logistics Provider
- EDI 204 (Motor Carrier Load Tender)

#### From Logistics Provider → SERMACROPS
- EDI 990 (Response to Load Tender)

---

### 3. API Design

All inbound EDI documents should be received via:
POST https://api.sermacrops/edi
Requirements:
- Accept raw EDI (ANSI X12) payloads
- Identify transaction type (e.g., 850, 855, etc.)
- Parse and validate structure
- Route to appropriate handler

---

### 4. System Architecture

Design modular components:
- `AS2Client` → Handles sending/receiving AS2 messages
- `EDIParser` → Converts ANSI X12 ↔ JSON
- `TransactionHandlers` → One handler per EDI type
- `Router` → Directs incoming EDI to correct handler
- `Logger` → Logs all transactions
- `Config Manager` → Handles partner settings (IDs, URLs, certs)

---

### 5. Functional Flow

Example:
1. Coffee Shop sends EDI 850
2. When verified/ accepted manually, System parses PO
3. System responds with EDI 855
4. System generates EDI 850 to Raw Materials Supplier
5. Supplier responds with 855, 856, 810
6. System processes incoming supplier documents
7. System sends 204 to Logistics
8. Logistics responds with 990
9. System sends ASN (856) and Invoice (810) to Coffee Shop
